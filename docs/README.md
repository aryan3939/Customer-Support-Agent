# `docs/` — Project Documentation

Comprehensive documentation for the Customer Support Agent project.
Start with `HOW_TO_RUN.md` if you're setting up for the first time.

## Files

| File | What It Covers | When to Read |
|------|---------------|--------------|
| `HOW_TO_RUN.md` | Quick-start guide — setup, install, run | **First time setup** |
| `API_KEYS_SETUP.md` | Step-by-step guide for all API keys (Google, Groq, LangSmith, Supabase) | **First time setup** |
| `PROJECT_STRUCTURE.md` | Complete file/folder map with architecture diagrams and data flow | **Understanding the codebase** |
| `01_environment_setup.md` | Detailed environment setup (Python, Node.js, Supabase) | **Deep dive into setup** |
| `02_project_setup.md` | Project initialization, folder structure decisions | **Understanding design choices** |
| `03_database_models.md` | ERD and explanation of all SQLAlchemy models | **Working with the database** |
| `04_core_agent.md` | LangGraph workflow design, node descriptions, edge conditions | **Understanding the AI agent** |
| `05_api_routes.md` | REST API endpoint documentation | **Working with the API** |

## Reading Order

1. `HOW_TO_RUN.md` — get it running locally
2. `API_KEYS_SETUP.md` — configure all external services
3. `PROJECT_STRUCTURE.md` — understand the full codebase
4. `04_core_agent.md` — deep-dive into the AI agent
5. `03_database_models.md` — understand the data model
